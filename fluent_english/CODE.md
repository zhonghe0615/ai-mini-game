# 宪法级（通用规则，适用于所有任务）
1.在编写任何代码或修改创建任何文件之前，请先描述你的方案并且等待我的批准，如果需求不明确，在编写任何代码之前务必使问题得到澄清，另一方面，我提出任何issue的时候，首先定位根因，再讨论解决办法，不要立马做修改
2.如果一个任务需要修改超过3个文件或200行，就请先停下来，将其分解为更小的任务，再进行处理
3.编写代码后，列出可能出现的问题，并建议相应的测试用例来覆盖这些问题
4.当发现bug时，首先要编写一个能够reproduce这个bug的测试，然后不断修复它，直到测试通过为止
5.每次我纠正你之后，当你相信这类问题具有普适性(而不是很特定条件下才会出现的)，你就在CODE.md文件中添加一条新规则，避免同样的问题再次出现

6.可验证性（让你能相信每次对话都按本文件执行）：在本项目中进行任何方案说明或代码修改后，回复中必须**简要列出本次遵循的 CODE.md 条款编号**（如：§1 先方案后改代码、§2 未超 3 文件/200 行、§9 Capacitor 优先、§6 动到登录/注册则跑 test:run 等）。未涉及的条款可不列。这样你可以通过“是否每次都看到条款引用”来复查 AI 是否按规则执行

7.代码可维护性，我们始终拒绝巨大的代码文件，原则上一个文件下代码行数超过500行就需要按照结构拆分具体方法到新的文件。
